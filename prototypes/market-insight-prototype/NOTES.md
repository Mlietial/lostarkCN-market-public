# 市场观察台新交互原型（可删除）

问题：真实数据应该怎样组织，才能更快完成“发现变化、横向比较、查看详情”三个任务？

用户明确要求只做一个全新方向，因此本原型不再制作 A/B/C 切换器。

当前假设：

- 首页只呈现跨市场摘要和按幅度排序的变化清单。
- 用户已确认“变化清单”方向可用；现补充方向筛选、排序、整行打开和动态今日重点。
- 每个市场使用“可筛选列表 + 原地检查器”，避免列表、图表和详情在不同页面间跳转。
- 表格不使用微型 SVG 趋势，改用精确的日变化和 7 日变化；详情趋势使用 Canvas 并提供悬停读数。
- 礼包默认只看在售，内容估值、覆盖率和原始图片都在右侧检查器中查看。
- 礼包回本率统一使用百分比展示：100% 等于成本，超过 100% 表示可识别价值高于成本。
- 宝石另设独立行情终端，使用真实日价格点验证“自选列表 + 动态行情图 + 等级价差 + 异动记录”的高密度查看方式。
- 宝石图表将日波动柱叠加在价格图底部，以 0% 中线区分上涨与下跌并按区间最大波动缩放；最新价格使用右侧独立标签。
- 报价摘要在宽屏中放置于宝石名称与当前价格之间；右侧不再显示重复的等级价差表。
- 宝石终端采用更易读的中等字号，列表、图表刻度、辅助说明和右侧数据表保持同一层级增幅。
- 顶部宝石行情条保持为持久 DOM，不随主内容交互重建；循环使用两个等宽长分组，避免点击重置和宽屏循环露空。
- 顶部行情滚动条吸附在主导航下方，页面向下滚动时持续可见。

运行：在本目录执行 `npm start`，访问终端显示的地址。

最终结论：待用户体验后填写；确认后把有效信息架构重写到正式页面，并删除本目录。
